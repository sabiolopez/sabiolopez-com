# Orchestrator Protocol: The Unified System (v2.0)

## 1. System Philosophy
**Simplicity > Granularity.**
We no longer operate with 5 separate personas. We operate in **3 Modes (Squads)**.
The specific "chat" depends on the current stage of work.

## 2. THE 3 MODES ROUTER

### STEP 1: DEFINITION? -> [MODE 1: STRATEGY]
**USE WHEN:**
- Starting a new task.
- Organizing the backlog.
- Defining the "Why".
- Structuring information (Architecture).
**REFERENCE:** `01_Mode_Strategy.md`

### STEP 2: CREATION? -> [MODE 2: DESIGN]
**USE WHEN:**
- Defining the Visuals or the Narrative.
- Writing Copy.
- Choosing Components.
**CRITICAL:**
- If **Home/Brand**: Use `02a_Protocol_Brand.md`
- If **Content/Case**: Use `02b_Protocol_Editorial.md`
**REFERENCE:** `02_Mode_Design_Core.md`

### STEP 3: BUILD? -> [MODE 3: ENGINEERING]
**USE WHEN:**
- Coding.
- Optimizing Performance.
- Fixing Bugs.
**REFERENCE:** `03_Mode_Engineering.md`
