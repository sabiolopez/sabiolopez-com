# A Arquitetura das Decisões

No mundo do SaaS B2B, um botão raramente é apenas um botão. É a culminação de restrições técnicas, lógica de negócio e intenção do usuário.

## O Problema com a Estética Vazia
A maioria dos designers foca em tornar a interface limpa. Em sistemas complexos — pense em ERPs ou portais de CRM — "limpo" às vezes significa "esconder a verdade".

### Pensamento Sistêmico
Quando desenhamos para sistemas, devemos mapear cada decisão.
- **Entrada:** Dados brutos do usuário ou sistema.
- **Lógica:** O que acontece com esses dados?
- **Saída:** O resultado que leva o usuário para o próximo passo.

> Arquitetura não é apenas como parece; é como funciona.
